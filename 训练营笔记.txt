Linux常用命令
	1. tree
	2. ls
		a. 功能：查看当前目录下的文件和文件夹
		b. 语法：
			i. Ls
			ii. Ls     路径 
		c. 参数：
			i. -a：显示所有文件， 包括隐藏文件
			ii. -l：显示文件的详细信息
			iii. drwxrw-r--   1    yy   yy   4096   Apr   11   11:11   aa
				1) 第一个字符：文件类型
					a) 普通文件：-
						i) .txt    压缩包   可执行文件
					b) 目录：d
					c) 链接文件：l
					d) 套接字：s
					e) 字符设备：c
					f) 块设备：b
					g) 管道：p
				2) rwx   rw-     r--
					a) rwx：文件所有者的权限
					b) rw-：文件所属组的权限
					c) r--：其他人的权限
					d) 思考：文件夹的rwx的是什么权限？
				3) 1：硬链接计数
	3. Cd  --   切换目录
		a. Cd     目录
		b. 怎么进入家目录
			i. Cd    /home/yy
			ii. Cd
			iii. Cd   ~
	4. pwd
		a. Print   work   directory
		b. 打印工作路径
	5. mkdir --  创建文件夹
		a. 语法：
			i. mkdir    文件夹的名字
	6. vi   --  文本编辑器
		a. 语法：
			i. vi     文件名
				1) 如果文件已存在，就是打开并编辑
				2) 如果文件不存在，就是创建并编辑
	7. rmdir  --  删除文件夹
		a. 矮穷矬
	8. rm   --  删除
		a. 删除文件
			i. rm      文件名
		b. 删除文件夹
			i. rm     -r     文件夹名
	9. cp 
		a. cp     目标文件    路径
		b. 情况：
			i. File(存在)    cp    dir(路径)
				1) 把file复制到dir
			ii. file1(存在)     cp    file2(不存在)
				1) 创建文件file2，把file1的内容复制到file2
			iii. file1(存在)   cp   file2(存在)
				1) 把file1的内容覆盖到file2
			iv. Dir1(存在)   cp   dir2(存在)
				1) 把dir1复制到dir2中
			v. Dir1(存在)   cp   dir2(不存在)
				1) 创建dir2，把dir1中的内容复制到dir2
	10. mv   --   剪切或重命名
		a. mv     目标文件    路径
		b. 情况：
			i. File(存在)     mv    dir(路径)
				1) 把file剪切到dir
			ii. file1(存在)   mv    file2(不存在)
				1) 把file1改名为file2
			iii. file1(存在)   mv   file2(存在)
				1) 把file1的内容覆盖到file2，并且file1消失
			iv. dir1(存在)   mv   dir2(存在)
				1) 把dir1剪切到dir2
			v. dir1(存在)   mv   dir2(不存在)
				1) 把dir1重命名为dir2
	11. 查看文件内容 -- 了解内容
		a. Cat
			i. 语法：cat    文件名
			ii. 适合看小文件
		b. More
			i. 语法：more    文件名
			向下浏览一行
				回车键
			向下浏览一页
				空格键
			只能向下，不能向上
		c. Less
			i. 语法：less    文件名
			向下浏览一行
				回车键
			向下浏览一页
				空格键
			向上浏览一页
				pageup键
			退出
				q
		d. Head
		e. tail
	12. 软硬链接 ln
		a. 软连接(快捷方式)
			i. Ln      -s      文件名      软连接名称
		b. 硬链接
			i. Ln      文件名      硬链接名

	Github	
		
	把本机的SSH Key添加到github：在本地创建SSH KEY $ SSH-keygen -t rsa -C "邮箱"，找到SSH KEY所在地复制id_rsa.pub文件内容，到官网中添加钥匙，测试是否连接成功=SSH -T git@github.com
	创建版本库：
	1. 创建一个空文件作为版本库
	2. 在git bash 下切换到文件夹目录，然后输入创建文件初始化命令 git init

	把文件添加到版本库：

	1.创建一个文件放到仓库
	2.查看仓库状态
		$ git  status
	3.把文件添加到版本库
		$ git  add  文件名
	4.提交修改
		$ git  commit  -m  "信息"
	5.查看文件版本信息
		$ git  log


	时光穿梭：
	回到以前的版本
		$ git  reset  --hard  版本
		版本：
		HEAD    代表当前版本
		HEAD^   代表上一个版本
		HEAD^^  代表上上一个版本
		HEAD~100   代表上100个版本
		版本号  回到指定版本
$ git  reflog：查看所有历史版本信息

	撤销修改
	1.在未提交之前，撤销修改
		$ git  checkout  --  文件名
	2.提交到暂存区，撤销修改
		清除暂存区的文件
			$ git  reset  HEAD  文件名
		替换工作区文件
			$ git  checkout  --  文件名
	3.在文件已经被commit之后，撤销修改
		版本回退	

	删除文件
	      版本回退
		$ git  reset  --hard  HEAD
	替换工作区文件
		$ git  checkout  --  文件名		
	
	Github	
		
	把本机的SSH Key添加到github
	1.在本地创建SSH Key
		$ ssh-keygen  -t  rsa  -C  "邮箱"
	2.找到SSH Key所在的地方
		C://User/用户名/.ssh
	3.复制id_rsa.pub文件中的内容
	4.到官网中添加钥匙
	5.测试是否连接成功
		$ ssh  -T  git@github.com

	把本地仓库上传到远程
	1.需要在远程建立一个空的仓库
	2.复制远程仓库的https或者ssh
	3.把地址添加到本地
		$ git  remote  add   origin  你的远程地址
	4.把本地仓库推送到远程
		$ git  push  -u  origin  master
		
	把远程仓库克隆到本地
	$ git  clone  地址
	同学A上传一个仓库到远程，同学下载A同学的仓库到本地
	
	参加开源项目

	1. 寻找参加的开源项目，fork到自己的仓库
	2. 把自己fork的项目clone到本地 git clone 地址
	3. 修改项目并在本地commit
	4. 把修改好的项目push到自己的远程仓库 git push -u origin master
	5. 然后在github中pull给原作者

注：
在remote时如果出现以下提示
fatal: remote origin already exists.
则执行下列命令，删除origin
git remote rm origin

	分支管理

	1. 创建分支 git branch 分支名称
	2. 查看当前分支 git branch
	3. 切换分支 git checkout 分支名称
	4. 合并分支 git merge 分支名称
	5. 删除分支 git branch -d 分支名称

	远程下载分支并处理冲突

	1. 从远程下载分支 git fetch origin 远程仓库分支名:本地分支名
	2. 对比两个分支的区别 git log -p 分支1 分支2
	3. 合并分支，并手动处理，最后上传
