Git笔记
	什么是Git？
                
分布式版本控制系统

	分布式和集中式

	Git的安装

	创建版本库：
	1. 创建一个空文件作为版本库
	2. 在git bash 下切换到文件夹目录，然后输入创建文件初始化命令 git init

	把文件添加到版本库：

	1.创建一个文件放到仓库
	2.查看仓库状态
		$ git  status
	3.把文件添加到版本库
		$ git  add  文件名
	4.提交修改
		$ git  commit  -m  "信息"
	5.查看文件版本信息
		$ git  log


	时光穿梭：
	回到以前的版本
		$ git  reset  --hard  版本
		版本：
		HEAD    代表当前版本
		HEAD^   代表上一个版本
		HEAD^^  代表上上一个版本
		HEAD~100   代表上100个版本
		版本号  回到指定版本
$ git  reflog：查看所有历史版本信息

	撤销修改
	1.在未提交之前，撤销修改
		$ git  checkout  --  文件名
	2.提交到暂存区，撤销修改
		清除暂存区的文件
			$ git  reset  HEAD  文件名
		替换工作区文件
			$ git  checkout  --  文件名
	3.在文件已经被commit之后，撤销修改
		版本回退	

	删除文件
	版本回退
		$ git  reset  --hard  HEAD
	替换工作区文件
		$ git  checkout  --  文件名		
	
	Github	
		
	把本机的SSH Key添加到github
	1.在本地创建SSH Key
		$ ssh-keygen  -t  rsa  -C  "邮箱"
	2.找到SSH Key所在的地方
		C://User/用户名/.ssh
	3.复制id_rsa.pub文件中的内容
	4.到官网中添加钥匙
	5.测试是否连接成功
		$ ssh  -T  git@github.com

	把本地仓库上传到远程
	1.需要在远程建立一个空的仓库
	2.复制远程仓库的https或者ssh
	3.把地址添加到本地
		$ git  remote  add   origin  你的远程地址
	4.把本地仓库推送到远程
		$ git  push  -u  origin  master
		
	把远程仓库克隆到本地
	$ git  clone  地址
	同学A上传一个仓库到远程，同学下载A同学的仓库到本地
	
	参加开源项目

	1. 寻找参加的开源项目，fork到自己的仓库
	2. 把自己fork的项目clone到本地 git clone 地址
	3. 修改项目并在本地commit
	4. 把修改好的项目push到自己的远程仓库 git push -u origin master
	5. 然后在github中pull给原作者

注：
在remote时如果出现以下提示
fatal: remote origin already exists.
则执行下列命令，删除origin
git remote rm origin

	分支管理

	1. 创建分支 git branch 分支名称
	2. 查看当前分支 git branch
	3. 切换分支 git checkout 分支名称
	4. 合并分支 git merge 分支名称
	5. 删除分支 git branch -d 分支名称

	远程下载分支并处理冲突

	1. 从远程下载分支 git fetch origin 远程仓库分支名:本地分支名
	2. 对比两个分支的区别 git log -p 分支1 分支2
	3. 合并分支，并手动处理，最后上传
